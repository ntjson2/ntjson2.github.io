What I built

AssayChain is a REST API for mineral commodity data—USGS commodity prices, field studies, processing research, all structured and on-chain. The goal was to make mineral data trustworthy and accessible so that AI systems, enterprises, and researchers can build on top of it without wondering if the data's legit.

What's in it

  • Processing know-how: Gravity separation, cavitation, placer vs. hardrock, recovery rates—all the mineral processing domain knowledge baked into queryable data.
  • 14 API endpoints: Query commodity prices, mine production, processing methods, historical data. Three auth tiers so you pay based on what you use.
  • On-chain proof: Field data lives on IPFS, attestations go on-chain via EAS, so you can actually verify that the data hasn't been messed with.
  • $0.10 per call: Cheap enough that AI systems and enterprises can actually use it at scale. No rate limit surprises.
  • Machine-readable: JSON responses designed so LLMs and semantic search tools can actually work with the data, not just humans reading docs.

The stack

React frontend to browse the data visually. Node.js backend pulling from USGS and aggregating everything. Solidity contracts for the on-chain attestations. IPFS for distributed storage. Google Cloud Functions so it scales without me babysitting servers. Firestore for real-time pricing and state.

Why it matters

If you're building an AI system or analytics platform that needs mineral data, you want to trust it. AssayChain gives you verified, immutable data with a clean API. Agents can query it. Humans can browse it. Everything's on-chain so you can audit it.
